Adapted from slime's swank-presentations.lisp

*record-repl-results*

Non-nil means that REPL results are saved for later lookup.

clear-repl-results

[connection]

Forget the results of all previous REPL evaluations.

initialize

[connection]

Set up the connection to use slime presentations

lookup-presented-object

[_ id]

Retrieve the object corresponding to ID. The secondary value indicates the absence of an entry.

lookup-presented-object-or-lose

[_ id]
[id]

Get the result of the previous REPL evaluation with ID.

object-to-presentation-id

Store the mapping of objects to numeric identifiers

presentation-counter

identifier counter

presentation-id-to-object

Store the mapping of numeric identifiers to objects

save-presented-object

[object]

Save OBJECT and return the assigned id. If OBJECT was saved previously return the old id.
