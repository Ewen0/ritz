ritz.swank.debug documentation

Debugger interaction for swank

backtrace

[connection start end]

Create a backtrace for the specified frames. TODO: seperate out return message generation.

breakpoint-list

[connection]

Update the context with a list of breakpoints. The list is cached in the context to allow it to be retrieveb by index.

debugger-info-for-emacs

[connection start end]

Calculate debugger display information

disassemble-symbol

[context thread sym-ns sym-name]

Dissasemble a symbol var

execute-unless-inspect

[handler]

If the function has resolved (in the debug proxy) then execute it, otherwise pass it on.

launch-vm-with-swank

[{:keys [port announce log-level classpath], :as options}]

Launch and configure the vm for the debugee.

line-breakpoint

[connection namespace filename line]

Set a breakpoint at the specified line. Updates the vm-context in the connection.

remote-swank-port

[context]

Obtain the swank port on the remote vm

setup-debugee

[connection]

Forward info to the debuggee. This uses request id 0, which will be filtered from returning a reply to slime.

thread-list

[connection]

Provide a list of threads. The list is cached in the context to allow it to be retrieved by index.
