ritz.debugger.break documentation

Break information. Break information is maintained based on thread id. For each thread a stack of break level-info's is maintained (since a thread may have multiple breaks pending).

aborting-level?

[connection thread-id]

Predicate to check if the current break-level is being aborted.

break-abort-to-level!

[connection thread-id level]

Set abort to level.

break-drop-level!

[connection thread-id]

Drop a break level.

break-drop-levels!

[connection thread-id]

Drop a break level.

break-exception-message

[connection thread-id event]

Set abort to level.

break-exception-message!

[connection thread-id event exception-message]

Set abort to level.

break-level-add!

[connection thread-id level-info]

Add a new break level. This clears :abort-to-level to ensure consistency of the break stack.

break-level-info

[connection thread-id]

Obtain the current level. Returns [level-info level-number]

break-level-infos

[connection thread-id]

Obtain the all level-infos.

break-threads

[connection]

Return threads referenced by context

clear-abort-for-current-level

[connection thread-id]

Clear any abort for the current level

clear-aborts

[connection]

Clear any abort

remove-threads

[connection thread-ids]

Remove threads from context
